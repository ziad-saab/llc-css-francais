#hypertext markup language ##L'étape du contenu.
##HTML: Petit rappel HTML sert à définir le **contenu** et les balises HTML servent à décrire les différents types de contenu. **Title**: se trouve dans la section `<head>` de votre document HTML. Il sert à définir le titre de votre page. Il parait dans l'onglet de votre navigateur, dans les résultats de moteurs de recherche et dans les favoris/*bookmarks*. ```xml <title>Titre de ma page</title> ``` **Headings**: Il existe 6 niveaux de balise d'entête (*heading*) (`<h1>,<h2>,...,<h6>`) et elles suivent un ordre hiérarchique. `<h1>` represente le titre le plus important de votre page, il vaut mieux l'utiliser qu'une fois. Les autres peuvent être réutilisés dans une même page. Il est important d'utiliser les bonnes balises avec le contenu approprié. Ces balises ont un rôle d'identification seulement, pas de style. NB: Une balise, en anglais c'est un *tag*. ```xml <h1>Entête importante</h1> <h2>Entête moins importante que h1, mais importante quand même.</h2> ``` **Paragraphs**: Créé avec les balises `<p></p>`. ```xml <p>Je suis un paragraphe.</p> ``` **Lien**: requiers l'ajout d'un **attribut ** `href` pour préciser la direction du lien. ``` <a href="http://siteweb.com">lien</a> ``` **Listes**: il existe 3 types de liste: ordonnées (`ol`) pour une liste numérotée et non ordonnée (`ul`) pour une liste avec des puces. Les items des listes doivent être définis comme des éléments de liste par des (`li`). ```xml <ul> <li>Élément de liste non-ordonée</li> <li>Élément de liste non-ordonée</li> </ul> <ol> <li>Élément de liste ordonée</li> <li>Élément de liste ordonée</li> </ol> ``` Il existe aussi les liste de définitions (`dl`) où on peut définir des éléments (`dt`) et leurs définitions (`dd`). ```xml <dl> <dt>Titre d'un element</dt> <dd>Définition de l'élément</dd> </dl> ``` **Images**: comme les liens, cette balise requiert l'usage d'un attribut `src`, cette fois, pour définir la source de l'image ajoutée. On peut utiliser des .jpg, des .jpeg, .png et des .gif . ``` <img src="chemin/vers/image.jpg"> ```

#Cascading Style Sheets ##L'étape du style !
##CSS, en bref Le HTML définit le **contenu**, le CSS définit l'apparence, le **style**. **Sélecteur/ (*Selectors*):** Utilisé pour déterminer quel(s) élément(s) nous allons styliser. Vous pouvez cibler une balise HTML, une classe ou un ID (on va le voir plus tard). **Bloc de déclaration/ (*Declaration block*):** C'est une liste de déclarations (règles à suivre) entourées d'accolades. **Règle ou déclaration/ (*Declarations*)** Elles sont les règles à suivre. Elles sont toujours composées d'une propriété et d'une valeur. ```css selector { property: value; } ``` **Propriété/ (*Properties*))** Détermine le type de style appliqué à l'élément (ex: color). **Valeur/ (*Values*)** Sont spécifiques à la propriété. Les déclarations doivent toujours se terminer par un (`;`) pour indiquer que l'instruction est complète. ![](framework/img/workshop/css.png)
##Commentaires Les commentaires en CSS sont écrits entre des barres obliques et des astérisques et doivent se clore dans l'ordre inverse. Les commentaires servent évidemment à laisser... des commentaires. Ils peuvent être des notes pour vous et pour les autres, servir à organiser votre code en blocs, ou encore permettre de *commenter* votre code (en cacher des portions, pour plus tard). ``` p { color: #cc0066; /* magenta */ } /* STYLE POUR LES ENTÊTES */ header { width: 80%; } /* À garder pour plus tard... .slider { width: 100px; background: red; } */ ```
##Les attributs Class & ID Quand les **sélecteurs** sont trop généraux (comme lorsqu'on cible des balises HTML), le HTML nous permet de mieux identifier nos éléments de 2 façons différentes : l'ajout d' **attributs** `class` qui peuvent être utilisés plusieurs fois et `id` qui peuvent être utilisés une seule fois par page. ``` /* targets ALL parapraphs */ p { color: blue; } /* s'applique seulement aux éléments qui portent cette classe, définie par un point. */ .danger { color: red; } /* s'applique seulement aux éléments qui portent cet id, définit par ce symbole. */ #danger { border: 1px solid red; } ``` ```xml <p>Texte régulier.</p> <p class="danger">Message d'erreur en rouge !!</p> <div id="danger">une bordure rouge.</div> ``` NB: la bonne pratique est d'éviter d'utiliser des id dans son CSS parce que ce sont des règles qui seront difficiles à gérer ensuite.
##class & id : convention dans les noms Lorsqu'on choisi un nom de `class` ou d' `id`, on utilise un nom descriptif et sans espace. Des tirets (*hyphens*) (-), des barres de soulignements (*underscores*) (_) et les CamelCase sont de mise. Le CSS est aussi sensible à la casse, donc bien que nos 3 exemples soient semblables le CSS les lira différemment. Il est important (pour votre santé mentale) de choisir une convention et de s'y tenir. ``` class="content-wrapper" class="content_wrapper" class="contentWrapper" ```
##CSS: les valeurs de couleurs **Mots clés** - Utiliser le réel nom de la couleur de base **en anglais, toujours**(red, green, blue, pink, etc). Il y en a même de surprenants comme "papayaWhip" et "paleGoldenrod"! **RGB (red-green-blue)** - Utiliser cette combinaison de 3 valeurs numériques entre 0 et 255. 0 est le noir et 255 est le blanc. Voir ci-dessous pour la syntaxe adéquate. **hex code** - En partant du symbole (`#`) suivi d'un code hexadécimal, soit 6 caractère, lettres et chiffres, entre 0-9 et A-F. ``` /* Voici différentes façons d'écrire la même couleur */ p { background: firebrick; background: #B22222; background: rgb(178,34,34); } ``` **Références pour les couleurs:** [http://www.colorpicker.com/](http://www.colorpicker.com/) [http://colours.neilorangepeel.com/](http://colours.neilorangepeel.com/) [http://www.colourlovers.com/](http://www.colourlovers.com/) [http://flatuicolors.com/](http://flatuicolors.com/) [http://color.hailpixel.com/](http://color.hailpixel.com/) [http://wesandersonpalettes.tumblr.com/](http://wesandersonpalettes.tumblr.com/)

##Inclure le CSS Il existe 3 différentes façons d'inclure (aussi appelé *referencing*) du CSS dans une page HTML ###Sur une ligne/ (*Inline*) On peut ajouter du CSS directement sur n'importe quel élément HTML grâce à l'**attribut** `style`. ``` <p style="color: blue;">J'écris du CSS!</p> ``` Cette technique devrait être évitée, mais il est bon de savoir qu'elle existe au cas où en verriez quelque part. ###Interne/ (*Internal*) Utilisez les balises `<style>` à l'intérieur de l'élément `<head>`. ``` <head> <title>Titre de ma page</title> <meta charset="utf-8"> <style> h1 { font-size: 16px; } </style> </head> <body> <p>J'écris toujours du CSS !</p> </body> ``` Favorisez cette technique plutôt que la méthode *inline*, elle est plus efficace puisque si vous avez plusieurs pages à votre site, vous auriez à copier le style de chacun des éléments au fur et à mesure que vous voudriez les réutiliser. Vous auriez à dupliquer du code pour une mauvaise raison. ###Externe/ (*External*) Créez un lien vers une feuille de style. Cette méthode est la plus recommandée, car elle permet de réellement séparer votre CSS (style) de votre HTML (contenu). Le lien vers votre feuille de style est inclue (*referenced*) dans le `<head>` de chacune de vos pages où votre style est requis. Ainsi, vous n'aurez besoin que de mettre à jour votre fichier CSS en fonction des nouveaux éléments que vous aurez besoin de styliser. Pour inclure votre CSS dans l'élément `<head>`, vous devrez créer une balise `<lien>` avec 2 **attribus** `rel` et `href`. Tout comme lorsqu'on crée un lien normal, le `href` pointe le chemin où est situé le fichier. Même si ce n'est pas obligatoire, les fichiers CSS sont souvent sauvegardés dans un dossier nommé CSS dans un souci d'organisation. Si vous organisez vos dossiers, assurez-vous que vos liens sont créés en conséquence. L'attribut `rel` définit le type de fichier. ``` <head> <title>Page Title</title> <meta charset="utf-8"> <link rel="stylesheet" href="css/style.css"> </head> ``` **Indice de pro!** Raccourci Sublime Text : tapez "link" + touche 'tab'. ##La cascade Le CSS s'exécute du haut vers le bas (voilà d'où viens la « cascade »), alors soyez prudentes avec l'ordre de vos déclarations. Une déclaration qui vient *après* peut en modifier une autre située au-dessus d'elle. Dans cet exemple, quelle couleur sera `<h1>` ? ``` h1 { color: black; } h1 { color: blue; } ```
##Héritage et spécificité Une des grandes forces du CSS est qu'un élément enfant peut hériter du style de son élément parent. ``` <body> <h1>Mon Site web</h1><!-- enfant de body --> <p>Ceci est un paragraphe.</p><!-- frère de h1 --> </body> ``` ``` body { color: blue; font-family: Helvetica, Arial, sans-serif; } h1 { color: black; } ``` Dans l'exemple ci-dessus, les propriétés CSS attribuées au sélecteur body font en sorte que tous ses éléments descendants auront un texte bleu et auront la police Helvetica. Cependant, quand un sélecteur plus spécifique (comme le h1, dans l'exemple) indique une nouvelle règle, c'est elle qui aura prédominance en fonction de sa spécificité.

#Le modèle de la boîte (*The Box Model*)
##Qu'est-ce que le *Box model* ? Le navigateur voit chaque élément HTML comme une boîte rectangulaire. Le modèle de la boîte CSS décrit la façon dont fonctionnent les styles CSS de taille et d'espacement. Comprendre le modèle de la boite est fondamental pour comprendre comment positionner des éléments en CSS pour recréer la structure que vous désirez. C'est une des parties plus complexes du CSS, ne vous en faites pas ! Le CSS utilise 5 propriétés pour déterminer la taille et l'espacement des boîtes. * `margin` - ajouter/enlever de l'espace *en dehors* de l'élément * `padding` - ajouter/enlever de l'espace *à l'intérieur* de l'élément * `border` - ajoute une bordure *en dehors* de l'élément * `width` - peut être défini à une certaine taille * `height` - peut être défini à une certaine taille

*Padding* et *Margin* (rembourrage et marges)

Le padding et les margin peuvent êtres déclarées en utilisant 1 ou 4 nombres. Les valeurs peuvent en utilisant différentes unités de mesure, mais les pixels et les pourcentages sont les plus communs. Définir la valeur à 0 va enlever tout espace mis par défault sur l'élément HTML. Toute valeur positive va ajouter de l'espace.

  • 4 valeurs signifient haut, droit, bas, gauche
  • 3 valeurs signifient haut, droit & gauche, bas
  • 2 valeurs signifient haut & bas, droite & gauche
  • 1 valeur signifie le même nombre d'espaces de chaque côté.

Changez quelques valeurs pour voir la différence entre les marges et le *padding*. Essayez avec 1 à 4 valeurs différentes. Aussi, essayez d'inspecter l'élément avec votre outil de développement dans Chrome.

##*Padding* et *Margin* (suite) La diapo précédente utilise la méthode la plus concise de déclarer du *padding* ou des *margins*. Vous pouvez aussi déclarer ces propriétés en parlant d'un côté en particulier. ``` padding-top: 10px; padding-right: 10px; padding-bottom: 10px; padding-left: 10px; ``` ``` margin-top: 10px; margin-right: 10px; margin-bottom: 10px; margin-left: 10px; ```
##Les bordures (*Border*) La propriété *border* se situe entre les *margins* et le *padding* et crée une ligne autour de l'élément. Sa syntaxe lui permet aussi de se déclarer de différentes façons. ``` /* Version longue */ border-width: 2px; border-style: solid; border-color: red; /* Version courte */ border: 2px solid red; ``` Comme pour *padding* et *margin*, un côté à la fois peut être spécifié. ``` /* Version longue */ border-top-width: 2px; border-top-style: solid; border-top-color: red; /* Version courte */ border-top: 2px solid red; ```

##Exercice pratique: Les images d'arrière-plan Jusqu'à maintenant, nous avons utilisé `background` pour les couleurs, mais la même propriété peut être utilisée pour régler une image d'arrière-plan. ``` background-image: url(path/to/file.jpg); /* longhand */ background: url(path/to/file.jpg);/* shorthand */ ``` Étant donné que votre fichier CSS est à l'*intérieur* d'un dossier, utilisez `../` pour *sortir* du dossier pour accéder au fichier d'images. Maintenant, ajoutons une image d'arrière-plan au `body` de notre page. Vous pouvez choisir une image parmi celles situées dans le dossier **projet > images** ou ajouter une image de votre choix. ``` background: url(../images/background-1.jpg); ``` Voyez comment l'image se répète? Réglons cela tout de suite.
##Exercice pratique: Les images d'arrière-plan (suite) Pour empêcher l'image de se répéter, utilisez `background-repeat` ou utilisez la version courte en ajoutant votre valeur à la propriété `background`. ``` /* version courte */ background: url(../images/background-1.jpg) no-repeat; /* version longue */ background-image: url(../images/background-1.jpg); background-repeat: no-repeat; ``` Maintenant, l'image ne prend pas toute la page mais, avant de modifier ceci, essayons d'ajouter un autre style. Ajoutez "fixed" à la version courte de la propriété `background`. Allez voir votre page et faites-la défiler. Que ce passe-t-il avec l'image? ``` /* version courte */ background: url(../images/background-1.jpg) no-repeat fixed; /* version longue */ background-image: url(../images/background-1.jpg); background-repeat: no-repeat; background-attachment: fixed; ```
##Taille de l'arrière-plan CSS3 a introduit la propriété `background-size` qui nous permet non seulement de changer la taille de l'image mais aussi de rendre l'image d'arrière-plan fluide et flexible. La syntaxe est: ``` background-size: width height; ``` La valeur par défaut pour la `width` et le `height` est `auto` et garde les dimensions originales de l'image. Si seulement une valeur est définie, la valeur sera considérée comme étant celle de la `width`. Le `height` sera réglé à `auto` et l'image maintiendra ces proportions initiales. Il y a plusieurs façons de définir les dimensions en utilisant px, pourcentages, mots-clés, etc. `background-size` est la version longue de la propriété. Pour l'inclure dans la version courte `background`, il *doit* être inclus après `background-position`, séparé par le caractère `/`. ``` header { background-image: url(../images/background-1.jpg); background-repeat: no-repeat; background-attachment: fixed; background-position: 50%; background-size: 100%; } ``` ``` header { background: url(../images/background-1.jpg) no-repeat fixed 50% / 100%; } ``` Si aucune valeur pour `background-position` n'est donnée, `background-size` ne fonctionnera pas dans la version courte. Donc, dans ce cas, il peut être plus facile de ne pas faire d'erreur en utilisant la version longue de la propriété. ``` header { background: url(../images/background-1.jpg) no-repeat fixed 50%; background-size: 100%; } ```
##Exercice pratique: Background-size Tout semble parfait... jusqu'à ce que vous diminuez la taille de l'écran. C'est à ce moment qu'utiliser le mot-clé `cover` fonctionnera mieux. `cover` ajuste l'image pour qu'elle remplisse le contenant en entier mais, si elle a des proportions différentes que celles de l'écran, l'image sera rognée. Changeons le `background-size` à `cover`. Votre code devrait ressembler à ceci: ``` background: url(../images/background-2.jpg) no-repeat fixed 50%; background-size: cover; ``` Et votre page devrait ressembler à l'[exemple 6](project/examples/6-background.html). **Ressource additionnelle**: https://developer.mozilla.org/en/docs/Web/CSS/background
##La propriété float N'oubliez pas, les éléments HTML bloc comme `<p>`, `<div>` and `<section>` commencent automatiquement sur une nouvelle ligne et se positionnent un par-dessus l'autre. Pour que deux blocs s'alignent un à côté de l'autre, `float` peut être utilisé. Les floats CSS peuvent flotter à gauche ou à droite. **Cependant**, nous brisons le *flux d'empilement normal* en mettant un élément en `float`. Tous les éléments *suivants* celui que nous avons mis en `float` essaieront de se déplacer à côté de lui ou n'apparaîtront pas comme prévu, car nous avons perturbé le flux d'empilement normal. Le contenant parent s'effondrera aussi. **Aussi**, si aucune largeur n'est définie, l'élément flottera, mais la largeur sera automatiquement la largeur de son contenu.

C'est une blague! On peut régler tout ça.

Float, Clear and Clearfix

D'abord, nous devons le briser, et ensuite le réparer. Un des concepts CSS qui portent le plus à confusion est à quel moment utiliser clear:both et la classe .clearfix. Règle d'or: si un élément suit l'élément mis en `float`, appliquez clear: both à cet élément. Si tous les éléments doivent être mis en `float`, appliquez la classe .clearfix à l'élément parent.

##Les sélecteurs enfant **Sélecteur enfant** - Correspond à n'importe quel élément *descendant* de l'élément déclaré avant lui. ```css p a { /* cible seulement les balises a imbriquées dans une balise p */ } .classname a { /* cible seulement les balises a imbriquées dans n'importe quel élément auquel nous avons attribué la classe classname */ } ``` **Truc de pro:** Comme dans le film Inception, n'allez jamais à plus de trois niveaux de profondeur! ``` nav ul li a span { /* NOOOOOOOOOOO */ } ``` N'hésitez pas à sauter un niveau, mais soyez conscients des règles de précision. Gradients

background peut aussi être utilisé pour créer un gradient. Utiliser les gradients CSS plutôt qu'une image améliore la flexibilité et la performance. Les gradients sont généralement une couleur qui se fond dans une autre. Avec le CSS, vous pouvez contrôler la direction du fondu, les couleurs ainsi qu'à quel endroit le changement se produit.

Les gradients sont réglés à l'aide de la propriété background-image. La version courte background fonctionnera aussi.

Pour régler les valeurs de couleur de début et de fin, utilisez linear-gradient(color, color). L'axe par défaut est du haut vers le bas, cependant, il est aussi possible de changer la direction de l'axe. Pour changer la direction de gauche à droite, donnez un paramètre additionnel au début de linear-gradient() en commençant par "to" suivi de la direction. (ex. "to right")

background-image: linear-gradient(to right, red, blue);

Dans l'exemple suivant, essayez de régler les couleurs de départ et de fin. Ensuite, essayez de changer la direction.

Voir plus bas pour le code final, mais seulement si vous êtes bloqué! ``` .skills { background: linear-gradient(rgba(0,0,0,0.8), rgba(0,0,0,0.8)),url(../images/background-4.jpg) no-repeat fixed; background-size: cover; } ``` ``` .skills .columns { width: 33.33%; float: left; } ``` ```xml <div class="skills-list clearfix"> <h3>List Option</h3> <div class="columns"><!--float this class! --> <h4>Best at:</h4> <ul> ... </ul> </div> ... </div> ```
##Le positionnement en CSS Parfois, floats et display ne font pas l'affaire. La propriété `position` peut être utilisé pour placer les éléments à des endroits spécifique sur une page sans affecter les éléments l'entourant comme `float` le fait. Il y a trois options: `relative`, `absolute` and `fixed`. Les éléments sont tous `static` par défaut. `position` est utilisé conjointement avec les propriétés offset: `top`, `right`, `bottom`, and `left`. Celles-ci sont utilisées pour identifier où un élément sera positionné. Aujourd'hui, nous utiliseront la position `fixed`. Pour en apprendre plus à propos de position, [allez ici](framework/extras-positioning.html).

Position: fixed

Quand nous positionnons un élément à fixed, il sera toujours visible dans la fenêtre et restera toujours au même endroit dans la page, même en défilant la page. Essayez d'ajouter une valeur top (ou n'importe quelle propriété offset: right, bottom, left).

##Icon fonts Les polices d'icônes sont une manière facile d'ajouter des images à votre site tout en ayant la flexibilité d'utiliser des propriétés de style CSS pour changer leur grandeur et leur couleur, par exemple. Après tout, ce *sont* des polices d'écriture. Il en existe plus d'une mais, aujourd'hui, nous utiliseront Font Awesome: [http://fortawesome.github.io/Font-Awesome/](http://fortawesome.github.io/Font-Awesome/) Sous **[Get Started](http://fortawesome.github.io/Font-Awesome/get-started/)**, il y a plusieurs options pour ajouter les fichier de *font*. Si vous utilisez le CDN, n'oubliez pas d'ajouter le "http" pour qu'il fonctionne quand vous utilisez votre page "localement" (sur votre ordinateur). Pour l'utiliser, [choisissez une icône](http://fortawesome.github.io/Font-Awesome/icons/) et copiez le code et les classes fournis et c'est tout! Tous les icônes de médias sociaux dans le *footer* du fichier index.html de votre projet sont incluses à l'aide d'une police d'icônes.
##Exercice pratique - Remettre à zéro le style des listes Ajoutez un peu de flair à votre liste de compétences en remplaçant la puce (*bullet*) par défaut par une icône. Pour enlever la puce par défaut, utilisez la propriété CSS et valeur, `list-style-type: none;`. Le `ul` est le sélecteur étant donné qu'il est utilisé pour créer la liste. Les listes ont aussi un padding par défaut pour créer l'indentation, donc il faut le mettre à zéro en utilisant la valeur 0. Étant donné que nous voulons enlever les puces de toutes les éléments de liste de cette section, nous pouvons toutes les cibler en même temps en utilisant un **sélecteur enfant**. Nous pouvons utiliser `.skill`, étant donné que c'est la classe pour la section au complet et `ul` pour cibler toutes les listes. ```css .skills ul { list-style-type: none; padding: 0; } ```
